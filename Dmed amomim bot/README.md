# DMED feedback bot

Telegram-бот и веб-админка для сбора отзывов по внедрению DMED.

## Что умеет

- Вход сотрудников по уникальной ссылке учреждения.
- Отзывы по сотрудникам-внедренцам и по процессу внедрения.
- ФИО отправителя и телефон: номер принимается только через Telegram-кнопку отправки собственного контакта.
- Антиспам: повторный отзыв по тому же объекту ограничен на 7 дней.
- Админ-команды в Telegram для учреждений, сотрудников, отзывов, статистики и экспорта.
- Сотрудника можно прикрепить к нескольким учреждениям.
- Полноценная веб-панель `/admin` с вкладками: учреждения, сотрудники, отзывы, аналитика, Telegram-админы.
- Отдельные отчеты по каждому учреждению и выбранному периоду.
- Отзывы во вкладке сайта фильтруются по учреждению, датам, типу, поиску и статусным вкладкам.

## Быстрый запуск

1. Скопируйте `.env.example` в `.env`.
2. Заполните `BOT_TOKEN`, `BOT_USERNAME`, `DATABASE_URL`, `ADMIN_IDS` и `WEB_ADMIN_TOKEN`.
3. Запустите сервисы:

```bash
docker compose up --build
```

После запуска:

- бот работает в Telegram;
- веб-админка доступна на `http://localhost:8000/admin`;
- миграции применяются автоматически при старте контейнера.

## Доступ админов

Добавление админов через пароль отключено.

Доступ к Telegram-админке получают:

- ID из переменной `ADMIN_IDS`;
- ID, добавленные действующим админом через `/add_admin telegram_id`;
- ID, добавленные через раздел «Администраторы Telegram» в веб-панели.

Веб-панель защищена отдельным токеном `WEB_ADMIN_TOKEN`.

## Основные команды

Пользователь:

- `/start TOKEN` - вход по ссылке учреждения.
- `/feedback` - начать отзыв.
- `/my_reviews` - история своих отзывов.
- `/settings` - выбор языка.
- `/help` - краткая инструкция.

Администратор:

- `/admin` - открыть админ-меню.
- `/add_institution` - создать учреждение и ссылку.
- `/add_employee` - добавить сотрудника-внедренца.
- `/add_admin telegram_id` - добавить администратора.
- `/institutions` - список учреждений.
- `/employees [institution_id]` - список сотрудников.
- `/reviews [institution_id]` - последние отзывы, можно отдельно по учреждению.
- `/stats [institution_id]` - статистика, можно отдельно по учреждению.
- `/export [xlsx|pdf] [institution_id]` - экспорт всех отзывов или отдельного учреждения.

## Веб-панель

Вкладки:

- `Учреждение` - создание, редактирование, архивирование, перевыпуск ссылок и быстрый переход к отчету учреждения.
- `Сотрудники` - создание сотрудников и привязка одного сотрудника сразу к нескольким учреждениям.
- `Отзывы` - фильтрация по учреждению, периоду, типу, поиску и статусам `Новые`, `Изучено`, `В работе`, `Закрыто`.
- `Аналитика` - статистика и экспорт PDF/Excel по всем данным или конкретному учреждению за выбранный период.
- `Администраторы Telegram` - добавление Telegram ID админов без пароля.

## Переменные окружения

- `BOT_TOKEN` - токен Telegram-бота.
- `BOT_USERNAME` - username бота без `@`.
- `DATABASE_URL` - строка подключения PostgreSQL, например `postgresql+asyncpg://dmed:dmed@postgres:5432/dmed_feedback`.
- `REDIS_URL` - Redis для FSM-состояний, можно оставить пустым для памяти процесса.
- `ADMIN_IDS` - Telegram ID админов через запятую.
- `WEB_ADMIN_TOKEN` - токен входа в веб-панель.
- `WEB_ENABLED` - включить веб-панель, по умолчанию `true`.
- `WEB_HOST` - хост веб-панели, по умолчанию `0.0.0.0`.
- `WEB_PORT` - порт веб-панели, по умолчанию `8000`.
- `TIMEZONE` - часовой пояс для дайджеста.
- `WEEKLY_DIGEST_ENABLED` - еженедельный дайджест админам.
- `AUTO_CREATE_TABLES` - аварийное создание таблиц без Alembic, обычно `false`.
